# 리얼티쓰 프론트엔드 채용과제 — 기능 요구사항

> 원본 문서: `docs/리얼티쓰 프론트엔드 채용과제.pdf`

---

## 1. 날씨 정보 표시

- Open API를 이용해 날씨 정보를 조회한다.
- 표시해야 할 날씨 정보:
  - 현재 기온
  - 당일 최저 기온
  - 당일 최고 기온
  - 시간대별 기온

---

## 2. 현재 위치 감지

- 앱 첫 진입 시 사용자의 현재 위치를 자동으로 감지한다.
- 감지된 위치의 날씨 정보를 메인 화면에 표시한다.

---

## 3. 장소 검색

| 항목    | 내용                                        |
| ------- | ------------------------------------------- |
| 검색 범위 | 대한민국 한정                                |
| 검색 단위 | 시·군·구·동 모든 단위 (기초자치단체, 구, 동 구분 없음)   |
| 검색 예시 | "서울특별시", "종로구", "청운동"                         |

### 동작 흐름

1. 검색어 입력 시 매칭되는 장소 리스트를 실시간으로 표시한다.
2. 사용자가 리스트에서 장소를 선택한다.
3. 선택된 장소의 날씨 정보를 조회하여 표시한다.
4. 해당 장소의 날씨 정보가 없는 경우 `"해당 장소의 정보가 제공되지 않습니다."` 문구를 UI에 표시한다.

### 데이터 소스

- 제공된 `korea_districts.json` 파일을 활용하여 장소 검색을 구현한다.
- JSON 데이터 구조 변경 사용 가능.

## 4. 즐겨찾기

### 기본 규칙

- 장소를 즐겨찾기에 **추가/삭제**할 수 있다.
- 즐겨찾기는 **최대 6개** 장소까지 추가 가능하다.
- 추가된 장소는 **카드 UI** 형태로 표시된다.

### 카드 표시 정보

- 장소 이름 (별칭)
- 현재 날씨 정보
- 당일 최저 기온 / 최고 기온

### 카드 인터랙션

| 액션          | 결과                                       |
| ------------- | ------------------------------------------ |
| 카드 이름 수정 | 즐겨찾기에 추가된 장소의 별칭을 수정할 수 있다 |
| 카드 클릭     | 해당 장소의 상세 페이지로 이동한다           |

### 상세 페이지

- [섹션 1](#1-날씨-정보-표시)에 명시된 **모든 날씨 정보**를 표시한다.
  - 현재 기온, 당일 최저/최고 기온, 시간대별 기온
