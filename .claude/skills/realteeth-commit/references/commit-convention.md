# 커밋 컨벤션

## 형식

```
tag(scope): 제목

- 세부 변경사항 1
- 세부 변경사항 2
```

- 제목은 한국어로 작성
- 변경이 단순하면 본문 생략 가능
- scope는 기능명 또는 컴포넌트명 (영어 kebab-case)

## 사용 tag

| tag | 사용 시점 |
|-----|----------|
| `feat` | 새로운 기능을 추가/구현했거나, API를 연동했을 때 |
| `fix` | 버그나 이슈를 고쳤을 때 |
| `style` | CSS나 퍼블리싱 코드를 수정할 때 |
| `refactor` | 로직이 아닌 코드를 수정했을 때 |
| `perf` | 코드의 성능을 높였거나 개선했을 때 |
| `chore` | 패키지 설치/업데이트, 설정 파일 변경 등 비즈니스 로직과 무관한 작업 |
| `revert` | 이전 커밋을 되돌릴 때 |
| `docs` | README, 주석 등 문서만 변경했을 때 |
| `test` | 테스트 코드 추가/수정 |

## scope 결정 규칙

1. **브랜치의 기능명을 우선** 사용한다. (`feature/car-edit` → `car-edit`)
2. 커밋 내용이 브랜치 기능과 직접 관련이 없으면 **컴포넌트명 또는 기능명**으로 대체한다.

| 브랜치 | 커밋 내용 | scope |
|--------|----------|-------|
| `feature/car-edit` | 차량 수정 폼 관련 변경 | `car-edit` |
| `feature/car-edit` | 공통 `toast` 스타일 수정 | `toast` |
| `feature/car-edit` | `PageHeader` 컴포넌트 수정 | `page-header` |

## 예시

```
feat(car-edit): 차량 정보 수정 페이지 추가
```

```
fix(car-edit): 필수값 미입력 시 저장 버튼 비활성화

- 필수 입력값 검증 로직 추가
- 중복 토스트 제거
```

```
style(toast): 토스트 메시지 스타일 수정
```

```
refactor(car): 차량 등록 및 수정 메시지 상수화

- CAR_MESSAGES 상수 모듈 추가
- 하드코딩된 문자열을 상수로 교체
```
