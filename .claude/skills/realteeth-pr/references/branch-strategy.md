# 브랜치 전략

## 브랜치 구조

```
main (배포 브랜치)
└── develop (개발 통합 브랜치)
    └── feature/(기능) ← develop에서 분기
```

## PR 흐름

### 1. 기능 개발 완료 시
- **소스**: `feature/(기능)`
- **타겟**: `develop`
- 관련 커밋 해시 또는 이슈 태깅

### 2. 모든 기능 구현 후 상용 배포 시
- **소스**: `develop`
- **타겟**: `main`
- 제목 끝에 **'상용 배포'** 필수
- develop에 병합했던 PR들을 태깅

## 타겟 브랜치 결정 방법

현재 브랜치 이름 패턴으로 판단:

| 현재 브랜치 | 타겟 브랜치 |
|------------|------------|
| `feature/(기능)` | `develop` |
| `develop` | `main` |
